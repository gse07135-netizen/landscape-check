## 1. 프로젝트 개요
---
### 한 줄 정의
조경 정량검토 웹 도구

### 제품 목표
조경 설계 및 엔지니어링 전문가들이 생태면적률 및 식재 법규 검토에 소요되는 비효율적인 수동 계산과 문서 생성 작업을 자동화하여 설계 효율성을 극대화하고, 인허가 과정에서의 규제 준수 어려움을 해소하여 프로젝트 진행 속도를 향상시키는 것을 목표로 합니다.

### 배경
현재 조경 설계 및 엔지니어링 분야에서는 생태면적률 산정이나 식재 법규 검토와 같은 정량적이고 복잡한 규제 준수 작업이 주로 수동 계산, 스프레드시트, 물리적 문서 참조를 통해 이루어져 비효율적이며 오류 발생 가능성이 높습니다. 특히 설계 변경 시마다 전체 검토 과정을 반복해야 하는 어려움이 있어, 이를 자동화하고 실시간 피드백을 제공하는 웹 기반 도구의 필요성이 증대되고 있습니다.

### 사용자 문제
조경 설계 및 엔지니어링 전문가들은 생태면적률 산정과 식재 법규 준수 여부를 확인하기 위해 복잡한 수동 계산과 여러 문서를 참조해야 합니다. 이 과정은 시간 소모적이고 오류 발생 가능성이 높으며, 설계 변경 시마다 전 과정을 반복해야 하는 비효율적인 작업으로 인해 프로젝트 진행에 어려움을 겪고 있습니다.

### 해결 방안
사용자가 프로젝트의 기본 정보, 공간 유형별 면적, 수목 정보를 웹에서 직접 입력하면, 생태면적률과 식재 법규 준수 여부를 실시간으로 자동 계산하여 적합/부적합 여부와 부족분을 즉시 확인할 수 있도록 합니다. 또한, 모든 검토 결과를 통합 리포트 및 엑셀 형식으로 손쉽게 출력할 수 있는 기능을 제공하여 문서 생성 효율을 대폭 향상시킵니다.

### 차별점
이 도구는 생태면적률 가중치 및 식재 법규 수치를 코드에 하드코딩하지 않고, 출처, 시행일, 버전 메타데이터를 포함한 별도 데이터(JSON)로 관리하여 규제 변화에 유연하게 대응하고 정확성을 보장합니다. 또한, 환경부 및 지자체 등 여러 기준 중 사용자가 적용 기준을 직접 선택할 수 있도록 하여 실제 업무 환경의 다양성을 지원하며, 입력값 변경 시 실시간으로 검토 결과를 업데이트하여 설계 반복 및 최적화 과정을 혁신적으로 단축시킵니다.

### 타겟 사용자
주요 타겟 사용자는 조경 설계사 및 조경 엔지니어링, 계획 분야 전문가입니다. 이들은 주로 주거 단지, 공원, 도시 기반 시설 등의 조경 설계를 담당하며, 생태면적률 산정, 식재 법규 준수 검토, 인허가 서류 작성 등의 업무를 수행합니다. 잦은 설계 변경과 규제 변화 속에서 빠르고 정확한 검토 결과를 필요로 하는 사용자들입니다.

### 사용자 시나리오
조경 설계 전문가는 새로운 프로젝트를 시작할 때, 웹 도구에 프로젝트명, 대지면적, 지자체, 용도지역, 사업유형 등 기본 정보를 입력하고 적용 기준(예: 서울시)을 선택합니다. 이어서 생태면적률 탭에서 공간 유형별 면적을 입력하면 실시간으로 생태면적률 적합 여부와 부족분을 확인합니다. 다음으로 식재 법규 탭에서 수목 목록을 입력하며 법정 조경면적, 최소 수목 수, 상록 비율 등 법규 준수 여부를 즉시 검토하고, 부족한 부분이 있으면 즉시 설계에 반영하여 최적화합니다. 모든 검토가 완료되면, 통합 리포트 탭에서 최종 리포트를 확인하고 PDF 및 엑셀 파일로 출력하여 인허가 서류로 활용합니다.

### 핵심 지표
월간 활성 사용자(MAU) 증가율, 생태면적률 및 식재 법규 검토 시간 단축률, 리포트 생성 건수, 사용자 피드백을 통한 기능 개선 및 만족도 지수, 지원 지자체 및 규제 기준 확장 수.

### 리스크
규제 기준 변경에 대한 빠른 업데이트 미흡 시 서비스 신뢰도 하락, 초기 데이터 입력 오류로 인한 검토 결과의 부정확성 문제, 타 지자체 및 추가 규제에 대한 확장성 미비, 사용자가 CAD/도면 연동 기능 부재에 대한 불만 제기 가능성, 유사 경쟁 도구 출현 시 차별점 유지의 어려움.

## 2. 생태면적률 자동 산정 및 검토
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

공간 유형별 면적 입력 시 가중치와 환산 면적을 자동으로 계산하고, 실시간으로 생태면적률, 목표치, 적합/부적합 여부, 부족분을 표시합니다.

**수용 기준:**
```
1. ○ 사용자가 공간 유형별 면적을 입력할 수 있어야 한다.
2. ○ 입력된 공간 유형에 따라 가중치와 환산 면적이 자동으로 계산되어야 한다.
3. ○ 총 환산면적, 대지면적, 생태면적률, 목표치, 적합/부적합 상태, 부족분이 실시간으로 표시되어야 한다.
4. ○ 산정표 미리보기 및 엑셀 내보내기 기능이 제공되어야 한다.
5. ○ v1(MVP) 개발 시 생태면적률 모듈이 가장 먼저 완성되어야 한다.
```


### 2.1 공간 유형별 면적 입력 테이블
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Web

녹지, 포장, 수경(물), 수목 베드 등 공간 유형별 면적을 입력할 수 있는 테이블을 제공합니다. 사용자는 각 공간 유형의 면적을 직접 입력할 수 있습니다.


#### 2.1.1 공간 유형 행 추가

사용자가 테이블에 새로운 공간 유형 행을 추가할 수 있는 버튼을 제공합니다. 최소 1개 이상의 공간 유형이 입력되어야 합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Web


#### 2.1.2 공간 유형 선택

각 행에서 공간 유형(녹지, 포장, 수경, 수목 베드 등)을 드롭다운으로 선택할 수 있도록 합니다. 선택된 공간 유형에 따라 해당 가중치가 자동으로 로드됩니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Web


#### 2.1.3 면적 입력 필드

각 공간 유형의 면적(m²)을 입력할 수 있는 숫자 입력 필드를 제공합니다. 소수점 이하 2자리까지 입력 가능하며, 0보다 큰 값만 허용합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Web


#### 2.1.4 행 삭제 기능

각 행의 삭제 버튼을 통해 해당 공간 유형 행을 제거할 수 있습니다. 마지막 남은 행은 삭제할 수 없도록 제한합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Web


#### 2.1.5 입력값 리셋 기능

입력된 모든 공간 유형별 면적을 초기화하고 기본 상태로 돌아갈 수 있는 리셋 버튼을 제공합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Web
